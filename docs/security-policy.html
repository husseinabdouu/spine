Security Policy

Last updated: March 2026 · This page is a placeholder — full policy coming soon.

Infrastructure

Spine is built on Supabase with row-level security (RLS) enabled on all user tables. No employee or system process can access your data without your authenticated session.

Data in transit

All communication between the Spine app and our servers is encrypted via TLS 1.3. Financial data fetched via Plaid is transmitted over Plaid's bank-grade encrypted channels.

Data at rest

All stored data is encrypted at rest using AES-256. Biometric data is processed on-device where Apple Health APIs permit, minimising the data that leaves your device.

Third-party integrations

Spine uses Plaid for read-only financial data access. Plaid is a SOC 2 Type II certified provider. We do not store your banking credentials.

Vulnerability disclosure

If you discover a security vulnerability, please report it responsibly to security@joinspine.io. We will respond within 48 hours.

This is a pre-launch placeholder. A full, legally reviewed Security Policy will be published before Spine's public release.
